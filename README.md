# PDF Просмотрщик

Веб-приложение для хранения и просмотра PDF-файлов:

- создание вложенных папок и загрузка PDF;
- просмотр PDF прямо в браузере (постраничная навигация, масштаб);
- **разделение экрана на 2 панели** — можно открыть, например, страницу 68
  одного документа в левой панели и страницу 198 в правой, и листать их
  независимо друг от друга;
- вход по email/паролю — файлы видны только владельцу аккаунта.

Стек: **Next.js (App Router) + Supabase (Auth, Database, Storage) + Vercel**.

---

## 1. Настройка Supabase

1. Зайдите на [supabase.com](https://supabase.com) и создайте новый проект.
2. Откройте **SQL Editor** → **New query**, скопируйте содержимое файла
   [`supabase/schema.sql`](supabase/schema.sql) и выполните его (кнопка Run).

   Этот скрипт создаст:
   - таблицы `folders` и `files` с поддержкой вложенных папок;
   - политики Row Level Security — каждый пользователь видит только свои
     данные;
   - приватный bucket `pdfs` в Storage и политики доступа к нему.

3. Откройте **Project Settings → API** и скопируйте:
   - `Project URL`
   - `anon public` ключ

   Они понадобятся для переменных окружения.

4. (Опционально) В **Authentication → Providers → Email** можно отключить
   подтверждение почты ("Confirm email"), если хотите сразу входить после
   регистрации без письма с подтверждением — удобно для личного
   использования.

## 2. Локальный запуск

```bash
npm install
cp .env.example .env.local
```

Заполните `.env.local` значениями из Supabase:

```
NEXT_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
```

Запуск:

```bash
npm run dev
```

Откройте http://localhost:3000 — вас перенаправит на страницу входа.
Нажмите **«Создать аккаунт»**, введите email и пароль (если подтверждение
почты включено — подтвердите письмо), затем войдите.

---

## 3. Деплой на Vercel через GitHub

1. Создайте репозиторий на GitHub и запушьте туда этот проект:

   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/<ваш-аккаунт>/<репозиторий>.git
   git push -u origin main
   ```

2. Зайдите на [vercel.com](https://vercel.com), нажмите **Add New → Project**
   и выберите этот репозиторий из GitHub.

3. В разделе **Environment Variables** добавьте те же переменные, что и в
   `.env.local`:

   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`

4. Нажмите **Deploy**. После завершения сборки сайт будет доступен по адресу
   вида `https://<ваш-проект>.vercel.app`.

5. В Supabase зайдите в **Authentication → URL Configuration** и добавьте
   адрес вашего сайта на Vercel в **Site URL** и **Redirect URLs** (это нужно
   для писем подтверждения почты).

## Как пользоваться

- **Новая папка** / **Загрузить PDF** — действуют относительно текущей
  выбранной папки (показана под кнопками, по умолчанию «Корень»). Кликните
  на папку, чтобы сделать её активной и развернуть.
- Клик по названию файла открывает его в **левой** панели.
- Кнопка со значком "колонки" у файла открывает его в **правой** панели и
  включает режим разделения экрана.
- Кнопка **«Разделить экран»** в верхней панели включает/выключает второй
  просмотрщик.
- В каждой панели — своя навигация: поле номера страницы, кнопки вперёд/назад
  и масштаб (+/-), независимые для левой и правой панели.
- Значки карандаша и корзины при наведении на папку/файл — переименование и
  удаление (удаление папки удаляет всё её содержимое, включая файлы в
  Storage).

## Структура проекта

```
app/
  layout.tsx          — корневой layout
  page.tsx            — главная страница (проверка авторизации + AppShell)
  login/              — страница входа/регистрации + server actions
components/
  AppShell.tsx        — общий каркас: шапка, сайдбар, область просмотра
  FileBrowser.tsx     — дерево папок и файлов, загрузка/CRUD
  FolderNode.tsx      — рекурсивный узел папки
  FileRow.tsx         — строка файла в дереве
  SplitViewer.tsx     — разделение экрана на 2 панели
  PdfViewerPane.tsx   — просмотрщик PDF (react-pdf), навигация и масштаб
lib/
  supabase/           — клиенты Supabase (browser/server/middleware)
  types.ts, files.ts  — типы и вспомогательные функции
supabase/
  schema.sql          — схема БД, RLS-политики, bucket для PDF
middleware.ts         — защита маршрутов (редирект на /login)
```
